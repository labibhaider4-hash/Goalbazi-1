Production deployment

Taking Goalbazi live

Follow these steps in order to deploy your app with PostgreSQL, Gunicorn, Railway hosting, a custom domain, and full user authentication.

Flask + Gunicorn PostgreSQL Railway Custom domain Session auth
1

Set up your project folder

Your final folder should have exactly these files. Copy everything from the ZIP you downloaded into one folder called goalbazi.

goalbazi/ ├── server.py # Flask app (PostgreSQL + auth) ├── index.html # Main dashboard ├── login.html # Login page ├── register.html # Registration page ├── app.js # Frontend JS ├── styles.css # Styles ├── requirements.txt # Python deps ├── Procfile # Gunicorn start command ├── railway.toml # Railway config ├── seed_prod.py # One-time DB setup ├── .gitignore └── assets/ └── goalbazi-logo.svg
2

Push to GitHub

Railway deploys from GitHub. Create a new repo and push your code.

# In your goalbazi/ folder: git init git add . git commit -m "Initial Goalbazi commit" # Create a new repo on github.com, then: git remote add origin https://github.com/YOUR_USERNAME/goalbazi.git git branch -M main git push -u origin main
Tip: Go to github.com/new, name it goalbazi, keep it private, and don't add a README (you already have your files).
3

Create a Railway project

Railway is the easiest way to host Python apps with a managed PostgreSQL database.

  • Go to railway.app and sign up (free)
  • Click New Project → Deploy from GitHub repo
  • Select your goalbazi repository
  • Railway will detect Python and start building automatically
Free tier: Railway gives you $5 of free credit per month — enough for a small app with low traffic. Upgrade to the $5/month Hobby plan for always-on hosting.
4

Add PostgreSQL database

In your Railway project dashboard:

  • Click + New → Database → Add PostgreSQL
  • Railway automatically creates a DATABASE_URL variable and makes it available to your app
  • No extra config needed — the server.py reads it automatically

After adding Postgres, go to your web service → Variables and add:

VariableValueStatus
DATABASE_URL Set automatically by Railway when you add Postgres Auto
SECRET_KEY A long random string — use python -c "import secrets; print(secrets.token_hex(32))" to generate one Required
5

Seed the production database

After your first deploy succeeds, run the seed script once to create all tables and add initial turf/league data. You do this from Railway's shell.

  • In Railway, open your web service → Settings → Shell (or use the Railway CLI)
  • Run:
python seed_prod.py

You'll see: ✅ Database seeded successfully.

Only run this once. The seed script is idempotent (it checks before inserting) but there's no need to run it again after the first time.
6

Buy and connect a custom domain

Railway gives you a free subdomain like goalbazi-production.up.railway.app. To use a custom domain like goalbazi.in:

Buy a domain:

  • Namecheap — good prices, easy DNS panel
  • GoDaddy — widely used
  • BigRock — popular in India, sells .in domains

Connect to Railway:

  • In Railway → your service → Settings → Networking → Custom Domain
  • Enter your domain (e.g. goalbazi.in) and click Add
  • Railway shows you a CNAME record to add
  • Go to your domain registrar's DNS panel and add the CNAME record Railway gives you
  • Wait 5–30 minutes for DNS to propagate
  • Railway handles HTTPS/SSL automatically — no extra setup needed
# Example DNS record to add at your registrar: Type: CNAME Name: @ (or www) Value: goalbazi-production.up.railway.app
7

Test your live site

Visit your Railway URL or custom domain. You should see the login page. Test the full flow:

  • Register a new account at /register
  • Log in at /login
  • Book a turf slot — verify it persists after refresh
  • Create a game and confirm attendance
  • Send a chat message in the lobby
  • Register a second account (use incognito) and join the same game
Something broken? In Railway → your service → Logs you can see real-time server output and error tracebacks to debug issues.

Ongoing deployments

Every time you push to main on GitHub, Railway automatically redeploys. Your workflow going forward is:

# Make changes locally, then: git add . git commit -m "Your change description" git push # Railway picks it up and redeploys in ~60 seconds
✓

What changed in the upgraded server.py

  • Flask replaces the raw ThreadingHTTPServer — proper routing, sessions, and WSGI support
  • PostgreSQL via psycopg2 replaces SQLite — concurrent writes, production-grade
  • Gunicorn serves the Flask app with multiple workers
  • Session-based auth — passwords hashed with SHA-256 + random salt, stored in Flask sessions (server-side cookies)
  • login_required decorator protects all API and page routes
  • Multi-user — CURRENT_USER_ID is gone; every request uses session["user_id"]
  • All SQL placeholders changed from ? (SQLite) to %s (psycopg2)
  • SECRET_KEY from environment variable for secure session signing
